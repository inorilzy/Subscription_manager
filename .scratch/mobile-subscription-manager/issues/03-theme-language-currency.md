# 03 — 个性化主题、语言与单一币种

**What to build:** 用户可以在 Settings 中选择浅色或深色外观、英文或简体中文以及一个应用级货币；选择会立即应用到已有流程并在重启后保留，同时任何可能重新解释现有金额的币种变化都会被明确说明。

**Blocked by:** 02 — 记录并持久化第一个月付订阅.

**Status:** done

- [x] Settings 提供可操作的 Light/Dark Appearance、English/简体中文 Language 和单一 Currency 设置，不显示账号、Pro、Security、Logout 或 WebDAV 占位操作。
- [x] 主题选择立即更新所有已有页面，并通过共享语义令牌保持 Vibrant Scout 层级、对比度和触感结构。
- [x] 当设备请求减少动态效果时，非必要的弹跳、抬升和大幅位移动画被关闭，同时操作状态仍清晰可见。
- [x] 英文和简体中文覆盖当前导航、空状态、表单、详情、验证、成功与错误消息，不出现面向用户的混合语言或原始翻译键。
- [x] 当前支持的金额在所有已有页面使用同一个货币格式化规则，SQLite 中的整数最小货币单位保持不变。
- [x] 当已有 Subscription 时更改 Currency，用户必须确认“数值不会按汇率转换，只会在新货币下重新解释”；取消确认不会改变设置或数据。
- [x] Theme、Language 和 Currency 设置持久化到本地，并在应用重新启动后恢复。
- [x] 自动化行为测试至少用英文和简体中文完成一次已有新增/查看流程，并验证主题、币种警告及重启持久化。
- [x] 缺失生产翻译键会使验证失败，而开发期回退语言为英文。
